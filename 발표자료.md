# 🌱 Plan-Ti (식물집사 일정관리 서비스)

> Spring Boot + MyBatis + JSP 기반 식물 물주기 관리 웹 애플리케이션

---

## 1. 프로그램 목적

**"바쁜 현대인을 위한 식물 물주기 캘린더"**

반려식물을 키우는 사람들이 겪는 가장 큰 어려움은 **"언제 물을 줬는지 기억하지 못하는 것"**입니다.

Plan-Ti는 다음과 같은 문제를 해결합니다:

| 문제 | 해결 |
|------|------|
| 물 준 날짜를 매번 기억하기 어려움 | ✅ 간편체크 버튼 한 번으로 기록 |
| 식물마다 물주기 간격이 다름 | ✅ 농촌진흥청 API에서 계절별 물주기 자동 적용 |
| 계절에 따라 물주기가 달라짐 | ✅ 봄/여름/가을/겨울 자동 전환 |
| 내 식물의 정확한 이름과 정보를 모름 | ✅ API 검색 + 도감 기능 |
| 식물 정보를 일일이 입력하기 번거로움 | ✅ 검색만 하면 자동 입력 |

### 3.2 테이블 명세

#### plant_info — API 식물 정보 캐시

| 컬럼 | 타입 | 설명 |
|------|------|------|
| cntnts_no | VARCHAR(50) PK | API 콘텐츠 번호 (수동: `MANUAL_타임스탬프`) |
| plant_name | VARCHAR(200) | 식물명 |
| plant_sci_name | VARCHAR(200) | 학명 |
| plant_eng_name | VARCHAR(200) | 영명 |
| distb_nm | VARCHAR(200) | 유통명 |
| image_url | VARCHAR(500) | API 이미지 URL |
| water_spring | VARCHAR(10) | 봄 물주기 코드 |
| water_summer | VARCHAR(10) | 여름 물주기 코드 |
| water_autumn | VARCHAR(10) | 가을 물주기 코드 |
| water_winter | VARCHAR(10) | 겨울 물주기 코드 |
| manage_level | VARCHAR(50) | 관리 난이도 코드 |
| growth_info | TEXT | 생육정보 |
| is_manual | CHAR(1) | 수동등록 여부 ('Y'/'N') |
| reg_date | DATETIME | 등록일시 |

#### my_plant — 사용자 등록 식물

| 컬럼 | 타입 | 설명 |
|------|------|------|
| plant_no | INT PK AUTO_INCREMENT | 식물 번호 |
| user_id | VARCHAR(50) | 사용자 ID (현재 'default') |
| cntnts_no | VARCHAR(50) FK | 참조 API 식물 |
| nickname | VARCHAR(100) | 사용자 별명 |
| photo_path | VARCHAR(500) | 업로드 사진 경로 |
| user_water_interval | INT NULL | 사용자 지정 물주기(일) |
| last_water_date | DATE | 마지막 물 준 날짜 |
| reg_date | DATETIME | 등록일시 |
| del_yn | CHAR(1) | 삭제 여부 ('Y'/'N') |

#### watering_log — 간편체크 기록

| 컬럼 | 타입 | 설명 |
|------|------|------|
| log_no | INT PK AUTO_INCREMENT | 로그 번호 |
| plant_no | INT FK | 식물 번호 |
| water_date | DATE | 물 준 날짜 |
| reg_date | DATETIME | 등록일시 |

### 3.3 관계 (Relationships)

- **plant_info 1 : N my_plant** — 하나의 API 식물 정보를 여러 사용자가 등록 가능
- **my_plant 1 : N watering_log** — 하나의 식물에 여러 번 물주기 기록
- **my_plant.del_yn = 'Y'** — 소프트 삭제 (실제 DELETE 없음)

### 5.2 레이어별 역할

#### Controller (Presentation Layer)
- HTTP 요청을 받아 View로 연결
- `@Controller` + `@ResponseBody` 조합으로 JSON API 제공 (검색, 간편체크)
- `Model`에 데이터를 담아 JSP로 전달

#### Service (Business Layer)
- **ApiService**: 농촌진흥청 API 호출 (RestTemplate), XML 응답 파싱 (Jackson XmlMapper)
- **CalendarService**: 계절 판별, 물주기 코드 → 일수 변환, 물필요 여부 계산
- **PlantService**: 식물 CRUD, API 캐시 관리, 물주기 간격 계산
- **WaterService**: 물주기 기록, 월별 통계

#### Repository (Data Access Layer)
- MyBatis Mapper 인터페이스 + XML 매핑
- `map-underscore-to-camel-case` 자동 변환
- `useGeneratedKeys`로 AUTO_INCREMENT 키 반환

### 5.3 주요 흐름

#### 식물 등록 흐름
```
1. 사용자 검색어 입력 → jQuery Ajax → /plants/search
2. ApiService.searchPlants() → 농촌진흥청 gardenList API 호출
3. XML 응답 → Jackson XmlMapper 파싱 → PlantInfoDTO 리스트
4. 검색 결과에서 식물 선택 → 폼 자동 채움
5. 등록 버튼 → POST /plants/register
6. PlantService.registerPlant()
   → cntntsNo로 중복 체크
   → 없으면 API 상세 조회 (gardenDtl) → plant_info INSERT
   → my_plant INSERT
7. 홈화면으로 redirect → 등록된 식물 표시
```

#### 물주기 체크 흐름
```
1. 홈화면 또는 내 식물 목록에서 [체크] 버튼 클릭
2. POST /water/check (Ajax)
3. WaterService.checkWater()
   → watering_log INSERT (오늘 날짜)
   → my_plant.last_water_date UPDATE (오늘 날짜)
4. location.reload() → 홈화면 새로고침
```

#### 물 필요 여부 계산 흐름
```
1. 캘린더 조회: CalendarService.needsWaterToday(lastWaterDate, interval)
2. lastWaterDate = 마지막 물 준 날짜
3. interval = 사용자 지정값
   → 없으면: API 계절별 물주기 코드 → 일수 변환
   → 없으면: DEFAULT_INTERVAL (7일)
4. daysSince = 오늘 - lastWaterDate
5. daysSince >= interval → "물 필요"
```
